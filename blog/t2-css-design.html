Classes vs. ID's

New concepts to share!

Monday, July 20, 2015

Let's start with ID's first. Generally, an Id can only be assigned to one element and each page you create can only have one element with that Id. As silly as it may sound, think on an ID as an individual person.

There is no other person like this one individual, which means that when we code, we can only assign an ID to one element because it is unique and can only be used for that one element.

Now, classes are a little different. They can be used on multiple elements and mulitple classes can be used on the same element as well. With classes, you can apply the same set of styles on different codes.

Both Id and classes can be used in a single element at the same time. Usually this is done to styel a comment on a page, but wanting to assign a unique value to that comment.
